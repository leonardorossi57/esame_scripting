\section{Simulation}

% \subsection{Overview of the interface}

The simulation, as mentioned above, is articulated in three steps, managed by different windows of the GUI. The first is the generation of the speckle fields, 
the second is the spatial filtering together with caculation of the interference patterns using the resulting filtered speckle fileds, the third is the analysis of 
the patterns and the calculation of the correlation function.

\subsection{Generation of the speckle fields} 
The interface allows to choose the desired size of the statistical sample as in figure \ref{specpar}.
\begin{figure}[!ht]
    \centering
    \includegraphics[width = \textwidth]{Img/speckle_param.png}
    \caption{Choice of parameters for step 1}
    \label{specpar}
\end{figure}

The fields $E_{\text{spec}}$ are created at distance $z$ from the source, summing spherical waves

\begin{equation}
  E(y) = \frac{e^{ikr(x) + \phi(x)}}{r(x)},  \qquad r(x) = \sqrt{(x - y)^2 + z^2},
\end{equation}

centered on points $x$ extracted rendomly in the source region, with random phases $\phi(x)$. 

\subsection{Spatial filtering}
The spatial filtering is done by computing a DFT of $E_{\text{spec}}$, multiplying by the pupil function of the filter (which in the experimental reality would be a 
rectangle function, though the options of the simulation also allow a gaussian) and then performing the inverse DFT obtaining $E_{\text{filt}}$. The DFT is supposed to represent a physical spatial 
filter built with two lenses and a slit in their focal plane: therefore the physical meaning of the spatial frequency $\tilde k$ (the independent variable of the DFT) is 
$\tilde k = k y / f$, where $k = 2\pi/\lambda$ is the wavenumber, $y$ the coordinate on the focal plane and $f$ the focal length of the lenses. The filtering width 
in frequency units is therefore $\tilde b = k b /f$. From \eqref{rect-corr} and \eqref{gauss-corr}, we see that the theoretical correlation function depends on the 
combination $\pi b s / \lambda f = k b s / 2 f = \tilde bs / 2$ (as expected, since implementation of the filtering procedure doesn't require a definition of $f$ nor of the 
physical dimension of the slit, the theoretical correlation function can be computed just from the filtering width in frequency units $\tilde b$). 

\subsection{Interference patterns}

The interface allows to choose the range of the slit separation and of the filtering width as in 
figure \ref{patpar}.
\begin{figure}[!ht]
    \centering
    \includegraphics[width = \textwidth]{Img/pattern_param.png}
    \caption{Choice of parameters for step 2}
    \label{patpar}
\end{figure}
The pattern is computed summing spherical waves weighted by the field $E_{\text{filt}}(x)$, i.e. contributes of the form
\begin{equation}
    E_{\text{filt}}(x) \, \frac{e^{ikr(x)}}{r(x)}, \quad r(x) = \sqrt{(x - y)^2 + z^2}
\end{equation}
for $x$ inside the interference slits. \\ 

The interference patterns produced look like figure \ref{patt}. In the figure, the modulating function of eqn. \eqref{theopatt} is also shown. Since this 
function is calculated neglecting the effects of partial coherence over each individual slit, the actual pattern has a lower intensity. \\

There are three main effects which in principle have to be taken into account, as can be seen in the figure: 
the modulation of the cosine (due to the finite width of the slits), the loss of visibility (due to the partial coherence between the field amplitudes at the two slits) 
and some other effect - which could be numerical or due to partial coherence across a single slit - causing a slight and inconsistent modification of the amplitude and the 
width of the pattern; this last effects are unaccounted for in the model, and are described by two additional parameters in the fit: a horizontal and a vertical dilaton 
constant.

\subsection{Data analysis}

The interface allows to visualize the interference patterns one by one, and to choose the method of analysis (mainly, how to calculate the modulation in order to remove it) as in 
figure \ref{data_int}.

\begin{figure}[!ht]
    \centering
    \includegraphics[width = 0.49\textwidth]{Img/data_int_1.png}
    \includegraphics[width = 0.49\textwidth]{Img/data_int_2.png}
    \caption{Data analysis interface}
    \label{data_int}
\end{figure}

The interface for data analysis allows to automatically compute the visibility of each interference pattern (previously generated and stored as csv files) as 
a function of both the width of the spectral density after the spatial filtering and the separation between the slits. For each fixed value of the filter width $\Delta k$, 
the resulting function is the absolute value of the normalized field correlation function, as per eqn. \eqref{vis_formula}: this allows to calculate the 
correlation length $\Delta l$ as the FWHM of the correlation function. The measured correlation function can be visualized for any value of the 
filter width as in fig. \ref{data_int}. It's then possible to plot the inverse relation between $\Delta l$ and $\Delta k$. \\

It's possible to choose two different metehods to estimate the modulation: either fit the upper profile with the modulating function in \eqref{theopatt} to the local maxima, 
or calculate numerically the convex hull of the local maxima (fig. \ref{an2} shows the convex hull). If a greater control over the analysis of a single pattern is necessary, 
the interface allows also to choose manually the starting parameters fo the fit of the modulation: in this case, the modulation is computed by jointly minimizing the 
$\chi^2$ for both the upper and the lower profile (respectively, to the local maxima and to the local minima of the pattern; fig. \ref{an1} shows the maxima and minima as 
computed by the script, fig. \ref{an3} shows the fit). \\

Normalizing the pattern by the modulation thus calculated, a nearly (co)sinusoidal pattern results (like in figure \ref{an4}); the visibility can then be calculated 
using eqn. \eqref{vis_formula}.

\subsection{Relation between \texorpdfstring{$\Delta l$}{dl} and \texorpdfstring{$\Delta k$}{dk}}

To each value of the width $\Delta k$ of the filtering slit there corresponds a different width $\Delta l$  of the filtered speckle field's correlation function. 
These two quantities are inversely proportional, and this can be directly verified from the simulation. This is just a well-known consequence of the Fourier transform 
relation between the spectral density and the correlation function (the Wiener-Khintchine theorem).
